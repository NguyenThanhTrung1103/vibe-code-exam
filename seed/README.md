# Test Dataset

This directory carries a portable snapshot of the **test environment**
data so anyone cloning the repo can stand up an identical instance.

> Test environment only. Do not point production at this dataset ŌĆö
> it carries the contents of the live LXC test DB and is not curated.

## Contents

| File | Format | Purpose |
|------|--------|---------|
| `test_dataset.sql` | PostgreSQL `--data-only --column-inserts` | Schema-free SQL: rows + sequence resets |

Tables included (data only ŌĆö schema comes from Alembic migrations):

| Table | Rows | Why |
|-------|------|-----|
| `providers` | 1 | Fortinet vendor |
| `product_versions` | 1 | FortiOS 7.4 |
| `courses` | 1 | NSE 4 track |
| `exams` | 1 | NSE 4 ŌĆö FortiGate Security (published) |
| `topics` | 0 | none seeded yet |
| `questions` | 259 | published question bank |
| `question_options` | 1076 | answer choices AŌĆōD for each question |
| `question_explanations` | 164 | overall explanations where available |
| `imports` | 4 | import history (XLSX uploads that produced the bank) |
| `import_items` | 320 | per-row import staging trail |
| `source_domains` | 5 | trust list (vendor docs domains) |

Tables intentionally **excluded**:

| Table | Reason |
|-------|--------|
| `users` | No real password hashes shipped. Create a test admin via `scripts/create_admin.py` after restore. |
| `sessions` | Per-host runtime state. |
| `attempts`, `attempt_answers` | Practice runs from the test environment ŌĆö restoring them would clutter; users can generate fresh attempts after restore. |
| `audit_logs` | Will be regenerated by application activity. |
| `question_reports` | None today; no value preserving. |
| `community_*` | Disabled feature in this release. |
| `ai_*`, `evidence_*`, `glossary_*` | Phase-2/3 placeholder tables; empty. |

## Restore

See `docs/restore-test-environment.md` for the full step-by-step. TL;DR:

```bash
sudo -u postgres pg_restore -d exam_platform_db --schema-only ...   # NOT used
# 1. Run migrations (creates schema from Alembic)
( cd /srv/exam-platform && .venv/bin/alembic upgrade head )
# 2. Load data
sudo -u postgres psql exam_platform_db -v ON_ERROR_STOP=1 -f seed/test_dataset.sql
```

## When to refresh this file

Re-export from the LXC after meaningful test-data changes
(new imports, vendor renames, etc.):

```bash
ssh exam-lxc 'sudo -u postgres pg_dump -d exam_platform_db \
    --data-only --column-inserts --no-owner --no-privileges --disable-triggers \
    -t providers -t product_versions -t courses -t exams -t topics \
    -t questions -t question_options -t question_explanations \
    -t imports -t import_items -t source_domains' \
  > seed/test_dataset.sql
git diff seed/test_dataset.sql   # review what changed before committing
```
